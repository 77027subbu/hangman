Hangman game using Apache Tapestry 5.4-beta.

Apache Tapestry is a component oriented framework for creating dynamic, robust, highly scalable web applications in Java. I am happy that I spent the time in learning this framework. I think it makes web development easy, fast and fun to work with. If you haven't tried it before, or have not tried their latest version 5 lately, I would highly recommend you to give it a try.

As a part of learning curve, I wanted to test my skills in using a Apache Tapestry 5 and so I started writing up a program to simulate the Hangman game. The beauty I liked in Tapestry is that I can write my code as if I'm writing sequences for a stage or a game or to be more precise like state diagrams.

I would like to divide the flow into 3 states or stages -

  • Initial Game Setup
  • User Guess
  • Win or Loss

Initial Game Setup

The user is presented with a screen like below with an initial hint of the word to be guessed.

User Guess

The user starts guessing the word. There are 2 states here -

The user guesses the letters correct and the letters start filling up -

The user guesses the letters wrong and the hangman image starts improving -

Win or Loss

The game completes and displays a score if the user guesses all the letters correct.

The user fails guessing the letters and hangman improves to the last state so the user is presented with the final screen without a score.
